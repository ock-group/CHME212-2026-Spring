Question 1 — ⏱ ~1 min#

Consider the following function definition:

def arrhenius(T, A=1.0, Ea=50000, R=8.314):
    import numpy as np
    return A * np.exp(-Ea / (R * T))

What is the return value of arrhenius(500) compared to arrhenius(500, A=2.0)?

(A) They return the same value because A does not affect the exponential term.
(B) arrhenius(500, A=2.0) returns exactly twice the value of arrhenius(500).
(C) arrhenius(500, A=2.0) returns four times the value of arrhenius(500).
(D) A TypeError is raised because A is a keyword-only argument.

Answer **(B)** — The function returns $A \cdot e^{-E_a/(RT)}$. Since `A` multiplies the entire expression, doubling `A` from 1.0 to 2.0 doubles the result. The other parameters are unchanged.

Question 2 — ⏱ ~1 min#

Given:

import numpy as np
T = np.array([300, 400, 500, 600, 700])   # Kelvin

Which expression correctly converts all temperatures to Celsius in a single vectorised operation?

(A) for i in range(len(T)): T[i] = T[i] - 273.15
(B) T_C = T - 273.15
(C) T_C = [t - 273.15 for t in T]
(D) T_C = np.subtract(T, [273.15, 273.15, 273.15, 273.15, 273.15])

Answer

(B) — NumPy broadcasting lets you subtract a scalar from every element at once with T - 273.15. It is the most concise and fastest option. (D) works but is unnecessarily verbose. (A) is an in-place loop — not vectorised. (C) produces a plain Python list, not a NumPy array.

Question 3 — ⏱ ~1 min#

A three-stream mixing problem yields the matrix equation \(\mathbf{A}\mathbf{x} = \mathbf{b}\), where \(\mathbf{x}\) contains unknown flow rates. The code below attempts to solve it:

import numpy as np
A = np.array([[1, 1, 1],
              [0.2, 0.5, 0.8],
              [0.1, 0.3, 0.6]])
b = np.array([100, 40, 20])
x = np.linalg.solve(A, b)

Before trusting the answer, which check is most important?

(A) Confirm np.linalg.det(A) != 0, indicating a unique solution exists.
(B) Confirm all elements of x are positive (physically meaningful flow rates).
(C) Verify by computing np.allclose(A @ x, b).
(D) All of the above are important, in roughly that order.

Answer

(D) — A non-zero determinant guarantees a unique solution exists (A). Physical validation checks sign/magnitude (B). Back-substitution A @ x ≈ b is the definitive numerical check (C). In practice (C) is the fastest single verification, but all three are good engineering habits.

Question 4 — ⏱ ~1 min#

You estimate \(\pi\) by randomly throwing \(N\) points into the unit square \([0,1]^2\) and counting how many fall inside the quarter-circle of radius 1. After 10,000 trials you get \(\hat{\pi} = 3.138\). What is the primary reason the estimate is not exactly \(\pi\)?

(A) The random number generator in Python is not truly random.
(B) Monte Carlo estimates have inherent statistical error that shrinks as \(1/\sqrt{N}\); 10,000 samples give roughly \(\pm 0.02\) uncertainty.
(C) The quarter-circle area formula was applied incorrectly.
(D) The unit square has an area of 1, so only the first quadrant can be sampled.

Answer

(B) — Monte Carlo integration has a statistical error of order \(1/\sqrt{N}\). For \(N = 10{,}000\), the standard error is about \(1/100 = 0.01\), so deviations like \(|3.138 - \pi| \approx 0.004\) are expected. This is a feature of the method, not a bug in the code or the formula.

Question 5 — ⏱ ~1 min#

Which of the following import statements lets you call sqrt(16) directly — without any module prefix?

(A) import math
(B) import math as m
(C) from math import sqrt
(D) from math import math.sqrt

Answer

(C) — from math import sqrt brings sqrt into the current namespace so you can call it as sqrt(16). Option (A) requires math.sqrt(16). Option (B) requires m.sqrt(16). Option (D) is a syntax error.

Question 6 — ⏱ ~1 min#

Which statement best describes the difference between a module and a package in Python?

(A) A module is a .py file containing Python code; a package is a directory that contains multiple modules (and optionally an __init__.py file).
(B) A module is installed with pip; a package is written by the user.
(C) A module can only hold functions; a package can also hold classes.
(D) There is no real difference — “module” and “package” are interchangeable terms.

Answer

(A) — A module is a single .py file. A package is a directory that groups related modules together and may optionally include an __init__.py file (in modern Python, this file is not required). Both modules and packages can contain functions, classes, and variables.

Question 7 — ⏱ ~1 min#

You create a file chem_utils.py in the same folder as your notebook, containing:

# chem_utils.py
GAS_CONSTANT = 8.314   # J / (mol·K)

def ideal_gas_volume(n, T, P):
    return n * GAS_CONSTANT * T / P

Which code block correctly imports only the function (not the constant) and calls it?

(A)

import chem_utils
volume = ideal_gas_volume(1, 300, 101325)

(B)

from chem_utils import ideal_gas_volume
volume = ideal_gas_volume(1, 300, 101325)

(C)

import chem_utils.ideal_gas_volume
volume = ideal_gas_volume(1, 300, 101325)

(D)

from chem_utils import *
volume = chem_utils.ideal_gas_volume(1, 300, 101325)
Answer

(B) — from chem_utils import ideal_gas_volume imports only the function and makes it available without a prefix. Option (A) works but requires the prefix chem_utils.ideal_gas_volume(...). Question 8 — ⏱ ~1 min#

A teammate’s code starts with:

import numpy as np
import matplotlib.pyplot as plt

Why is using aliases (np, plt) considered good practice?

(A) It makes the code run faster because shorter names are parsed more quickly.
(B) It prevents name collisions with built-in Python functions.
(C) It follows widely adopted community conventions, making code easier to read and share across projects.
(D) It is required by Python — you cannot use numpy or matplotlib.pyplot directly without an alias.

Answer

(C) — Aliases like np and plt are community-wide conventions (documented in the NumPy and Matplotlib official guides). Anyone familiar with scientific Python immediately recognises them. They do not affect runtime speed, are not required by Python, and do not prevent name collisions (that is handled by namespaces).

Question 9 — ⏱ ~1 min#

Consider:

import math

pi = 3.0   # user-defined variable

print(pi)          # line A
print(math.pi)     # line B

What do lines A and B print, respectively?

(A) Both print 3.141592653589793 because math.pi overwrites the local variable.
(B) Line A prints 3.0; line B prints 3.141592653589793.
(C) Line A raises a NameError because pi is shadowed by math.pi.
(D) Both print 3.0 because the local variable takes precedence everywhere.

Answer

(B) — import math does not overwrite names in the local namespace. The user-defined pi = 3.0 lives in the local/global scope, while math.pi lives inside the math module’s namespace. Using the math. prefix explicitly accesses the module’s version, so both names coexist without conflict.

Question 10 — ⏱ ~1 min#

You have 20 reactor yield measurements and want to compute the sample standard deviation. Which call is correct, and why?

(A) np.std(yields) — the default is always correct for experimental data.
(B) np.std(yields, ddof=1) — dividing by \(N-1\) corrects for the bias introduced by estimating the mean from the same data.
(C) np.std(yields, ddof=2) — you must account for two lost degrees of freedom when computing std from a sample.
(D) np.std(yields) / np.sqrt(20) — you need to normalize by the sample size.

Answer

(B) — NumPy’s default (ddof=0) divides by \(N\), giving the population standard deviation. For a sample drawn from a larger population, ddof=1 divides by \(N-1\) (Bessel’s correction), which produces an unbiased estimate of the true population variance. One degree of freedom is “used up” when the sample mean is estimated from the same data.
