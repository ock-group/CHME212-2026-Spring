Core Concepts to Watch Out For#

Ch 10 — NumPy#

  • A NumPy array evaluates operations on all elements at once (vectorized) — no for loop needed

  • np.array([1, 2, 3]) * 2 → [2, 4, 6]; the same on a Python list gives a repeated list, not element-wise math

  • Indexing: A[i, j] is row i, column j (both zero-based). A[1, 2] is the 2nd row, 3rd column.

Ch 11 — Linear Algebra & Mass Balances#

The core idea: any system of \(n\) linear equations in \(n\) unknowns can be written as \(A\mathbf{x} = \mathbf{b}\), where each row of \(A\) encodes the coefficients of one balance equation. The solution \(\mathbf{x} = A^{-1}\mathbf{b}\) exists and is unique as long as \(\det(A) \neq 0\) — i.e., the equations are independent.

Building the system from mass balances:

  • Every balance has the form \((flow\ in) - (flow\ out) = 0\)

  • The unknowns (flow rates, concentrations) become entries of \(\mathbf{x}\)

  • Their coefficients in each equation become the corresponding row of \(A\)

  • The right-hand side constants become \(\mathbf{b}\)

Verification: substitute \(\mathbf{x}\) back into the original equations — the residual \(A\mathbf{x} - \mathbf{b}\) must be \(\approx \mathbf{0}\). This directly tests whether the solution satisfies every balance simultaneously.

MAE (Mean Absolute Error) — the average magnitude of the errors, ignoring direction. Straightforward to interpret: “on average, my prediction is off by this much in the same units as the data.” Less sensitive to large outliers because errors are not squared.

MSE (Mean Squared Error) — squares each error before averaging, so large mistakes are penalized much more heavily than small ones. Has units of \(y^2\), which makes it hard to interpret directly, but it is mathematically convenient (smooth and differentiable everywhere).

RMSE (Root Mean Squared Error) — the square root of MSE, restoring the original units of \(y\). Comparable to MAE but more sensitive to large errors. A useful rule of thumb: if RMSE \(\gg\) MAE, there are a few large outliers dominating the error.

\(R^2\) (Coefficient of Determination) — measures how much of the variance in the data is explained by the model, relative to simply predicting the mean every time: $\(R^2 = 1 - \frac{\text{SS}_\text{res}}{\text{SS}_\text{tot}} = 1 - \frac{\sum(y_i - \hat y_i)^2}{\sum(y_i - \bar y)^2}\)$

  • \(R^2 = 1\): perfect fit — the model explains all variation

  • \(R^2 = 0\): the model does no better than predicting the mean \(\bar y\) for every point

  • \(R^2 < 0\): the model is worse than the mean (possible for a badly misspecified model)

Ch 13–14 — Statistics & Monte Carlo#

Why Monte Carlo? Two complementary reasons:

  1. The nominal calculation is blind to spread. Evaluating a model at the mean input (\(\tau = \mu\)) tells you nothing about what happens when inputs vary. The output distribution can have a long tail that causes many failures even when the nominal result passes the spec.

  2. Analytical uncertainty propagation is generally impossible. For a nonlinear function \(y = g(x_1, x_2, \ldots)\) with uncertain inputs, deriving the exact output distribution analytically requires solving integrals that have no closed form. Monte Carlo sidesteps this entirely — just sample the inputs, run the model, and collect the outputs.

Ch 15 — Numerical Integration & Differentiation#

Trapezoidal rule approximates the area under a curve by connecting adjacent data points with straight lines and summing the resulting trapezoid areas: $\(\int_a^b f(x)\,dx \approx \sum_{i=0}^{n-1} \frac{x_{i+1}-x_i}{2}\bigl(f_i + f_{i+1}\bigr)\)$

  • Error is \(O(h^2)\): halving the step size quarters the error

  • The straight-line segments cut corners off a curved function — the gap between the line and the curve is the truncation error; more points reduce this gap faster than linearly

Finite difference approximates a derivative from discrete data by replacing the limit with a ratio of finite differences:

Position

Formula

Accuracy

Interior

\(f'(x_i) \approx \dfrac{f(x_{i+1}) - f(x_{i-1})}{x_{i+1} - x_{i-1}}\) (central)

\(O(h^2)\)

Left end

\(f'(x_0) \approx \dfrac{f(x_1) - f(x_0)}{x_1 - x_0}\) (forward)

\(O(h)\)

Right end

\(f'(x_n) \approx \dfrac{f(x_n) - f(x_{n-1})}{x_n - x_{n-1}}\) (backward)

\(O(h)\)

Central difference is more accurate because it uses symmetric information on both sides of the point; endpoints are less accurate because they can only look in one direction.

Ch 16 — Root Finding & Newton’s Method#

Newton’s method derives from a first-order Taylor expansion — linearize \(f\) at the current guess and find where that tangent line crosses zero: $\(f(x) \approx f(x_n) + f'(x_n)(x - x_n) = 0 \quad\Longrightarrow\quad x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)}\)$

  • Quadratic convergence: the error roughly squares each step — correct decimal places double with every iteration, far faster than bisection’s linear halving

  • When it fails: if \(f'(x_n) \approx 0\) the tangent is nearly flat and the next iterate shoots far away; a poor initial guess can also send the method toward the wrong root

  • Convergence check: test \(|f(x_n)|\) or \(|x_{n+1} - x_n|\) — never \(|x_n|\) itself, which measures location not progress

Bisection is slower but guaranteed — it uses the Intermediate Value Theorem: if \(f(a) \cdot f(b) < 0\), a root must exist in \([a, b]\). Each step halves the bracket:

  • After \(n\) steps: error \(\leq (b-a)/2^n\); steps needed for tolerance \(\epsilon\): \(n \geq \log_2\!\left(\frac{b-a}{\epsilon}\right)\)

  • Linear convergence (\(O(1/2^n)\)) — predictable but slow compared to Newton’s quadratic convergence

Ch 17 — Symbolic Math (SymPy)#

  • Symbols are exact — \(\sqrt{2}\) stays \(\sqrt{2}\) until you call .evalf()

  • Key strength: derive analytical expressions (sp.diff, sp.integrate, sp.solve) that a numerical tool cannot

Ch 18 — ODEs & Euler’s Method#

Euler’s method derives from replacing the derivative with a forward finite difference — take one straight-line step in the direction of the current slope: $\(\frac{dy}{dt} \approx \frac{y_{n+1} - y_n}{h} = f(t_n, y_n) \quad\Longrightarrow\quad y_{n+1} = y_n + h\,f(t_n, y_n)\)$

  • 1st-order accurate (\(O(h)\)): halving \(h\) halves the global error — much slower improvement than the trapezoidal rule (\(O(h^2)\)) because Euler uses only the slope at the start of the step and ignores how the slope changes across the interval

  • The error accumulates step by step: each local approximation is off by \(O(h^2)\), and summing over \(\sim 1/h\) steps gives a global error of \(O(h)\)

Problem Type 1: Multiple Choice — Concept Questions#

These test whether you understand what a tool does, why it works, and what pitfall to avoid. No code needs to run in your head — reason from definitions.

How to approach them#

  1. Read carefully — one word often changes the answer (“halve”, “double”, “residual”, “iterate”)

  2. Eliminate obviously wrong answers first

  3. Check units / dimensions when in doubt

Q2. You run a Monte Carlo simulation with \(N = 10{,}000\) samples and find 400 failures. How many samples are needed to halve the width of the 95% confidence interval?

(A) 5,000    (B) 20,000    (C) 40,000    (D) 100,000

▶ Answer

(C) 40,000 — CI width \(\propto 1/\sqrt{N}\). To halve the width you need \(4\times\) the samples: \(4 \times 10{,}000 = 40{,}000\).

Q3. Which of the following statements about goodness-of-fit metrics is correct?

(A) \(R^2\) carries the same units as the predicted variable \(y\)
(B) MSE is always between 0 and 1, where 1 means a perfect fit
(C) MAE and RMSE both have the same units as \(y\), while \(R^2\) is dimensionless
(D) RMSE is dimensionless because it squares the errors before averaging

▶ Answer

(C) — MAE \(\left(=\frac{1}{n}\sum|y_i - \hat{y}_i|\right)\) and RMSE \(\left(=\sqrt{\frac{1}{n}\sum(y_i-\hat{y}_i)^2}\right)\) both retain the same units as \(y\). \(R^2\) is dimensionless (a ratio of variances) and lies in \([0, 1]\) for a well-fitted model. MSE has units of \(y^2\), not \(y\), and is not bounded to \([0, 1]\).

Q4. In the trapezoidal rule, suppose you keep the same integration range but reduce the step size from \(h\) to \(h/2\). How does the number of steps change, and by approximately what factor does the integration error decrease?

(A) Steps double; error decreases by 2
(B) Steps double; error decreases by 4
(C) Steps quadruple; error decreases by 4
(D) Steps quadruple; error decreases by 16

▶ Answer

(B) The number of steps is inversely proportional to \(h\), so halving \(h\) doubles the number of steps. Since the trapezoidal rule has error of order \(O(h^2)\), halving \(h\) reduces the error by \(2^2 = 4\).

Problem Type 2: Multiple Choice — Fix the Code#

Each snippet contains one conceptual bug. Your job: identify the mistake and pick the corrected version.

How to approach them#

  1. Read the code carefully — focus on the logic, not just syntax

  2. Mentally trace what the code actually does vs. what it should do

  3. The bug is almost always conceptual (wrong formula, wrong check, wrong argument order) — not a typo

Practice Questions — Fix the Code#

FC1. A student wants to find \(\sqrt{2}\) by solving \(f(x) = x^2 - 2 = 0\) with brentq. Running the code raises:

ValueError: f(a) and f(b) must have different signs

from scipy.optimize import brentq

def f(x):
    return x**2 - 2

root = brentq(f, 2, 3)   # <-- what is wrong?

(A) brentq cannot handle polynomial functions — use fsolve instead
(B) The tolerance is too tight; pass a more lenient xtol to fix the error
(C) The bracket endpoints both give positive values: \(f(2) = 2 > 0\) and \(f(3) = 7 > 0\) — there is no sign change, so brentq cannot locate a root
(D) The bracket \([2, 3]\) is too small to evaluate the correct answer

▶ Answer

(C) — brentq requires \(f(a)\) and \(f(b)\) to have opposite signs (the Intermediate Value Theorem sign-change condition). With \(f(x) = x^2 - 2\), the positive root is \(\sqrt{2} \approx 1.41\), which lies entirely outside \([2, 3]\):

f(2) = 4 - 2 =  2  (positive)
f(3) = 9 - 2 =  7  (positive)   ← same sign → ValueError

Always plot \(f(x)\) first to visually confirm a sign change before calling brentq:

import numpy as np, matplotlib.pyplot as plt
x = np.linspace(0, 3, 200)
plt.plot(x, x**2 - 2); plt.axhline(0, color='k'); plt.grid(); plt.show()

# Valid bracket: f(1) = -1 < 0, f(2) = 2 > 0
root = brentq(f, 1, 2)

FC2. A student’s Newton’s method loop converges too early. What is wrong?

x = 3.0
for i in range(10):
    x = x - f(x) / df(x)
    if abs(x) < 1e-6:       # convergence check
        break

(A) Should use while True instead of for i in range(10)
(B) The convergence check tests |x| (the iterate’s magnitude) instead of |f(x)| or |x_new - x_old| — near the root \(x = 2\), |x| is never small
(C) abs() should be np.abs()
(D) The update should add instead of subtract: x = x + f(x)/df(x)

▶ Answer

(B) — we should check whether the residual \(|f(x)|\) or the step size \(|x_{\text{new}} - x_{\text{old}}|\) is small — not whether the iterate itself is near zero. For \(f(x) = x^2 - 4\), the root is at \(x = 2\), so \(|x|\) is never less than \(10^{-6}\).

if abs(f(x)) < 1e-6:    # correct convergence check
    break

FC3. A student computes \(\Delta H = \int_{400}^{1000} C_p(T)\, dT\) from tabular data. What is the bug?

from scipy.integrate import trapezoid

T   = np.array([400, 500, 600, 700, 800, 900, 1000])  # K
Cp  = np.array([34.3, 35.2, 36.3, 37.5, 38.6, 39.7, 40.8])  # J/mol/K

delta_H = trapezoid(Cp)    # <-- is this right?

(A) Should use quad instead of trapezoid
(B) trapezoid(Cp) uses unit spacing (\(\Delta T = 1\) K) instead of the actual 100 K spacing — must pass T as the second argument
(C) Cp and T arrays must be the same length
(D) Should integrate T with respect to Cp

▶ Answer

(B) — without the second argument, trapezoid assumes \(\Delta x = 1\). Fix:

delta_H = trapezoid(Cp, T)   # correct: uses actual T spacing

Worked Example 3 — Linear Algebra Mass Balance (By Hand via Inverse Matrix)#

Three streams mix at a junction. Solve for the flow rates \(F_1, F_2, F_3\):

\[F_1 + F_2 + F_3 = 100\]
\[0.30\,F_1 + 0.50\,F_2 + 0.20\,F_3 = 35\]
\[0.10\,F_1 + 0.20\,F_2 + 0.40\,F_3 = 24\]

Write in matrix form \(A\mathbf{x} = \mathbf{b}\), so the solution is \(\mathbf{x} = A^{-1}\mathbf{b}\).

Step 1 — Inverse of a 3×3 matrix

For a \(3\times3\) matrix \(A\), the inverse is:

\[A^{-1} = \frac{1}{\det(A)}\,C^T\]

where \(C^T\) is the transpose of the cofactor matrix (also called the adjugate). Each cofactor \(C_{ij} = (-1)^{i+j} M_{ij}\), where \(M_{ij}\) is the \((i,j)\) minor — the determinant of the \(2\times2\) submatrix formed by deleting row \(i\) and column \(j\).

Step 2 — Compute \(\det(A)\) by cofactor expansion along row 1

\[\begin{split}A = \begin{bmatrix} 1 & 1 & 1 \\ 0.30 & 0.50 & 0.20 \\ 0.10 & 0.20 & 0.40 \end{bmatrix}\end{split}\]
\[\begin{split}\det(A) = 1\cdot\det\begin{bmatrix}0.50&0.20\\0.20&0.40\end{bmatrix} -1\cdot\det\begin{bmatrix}0.30&0.20\\0.10&0.40\end{bmatrix} +1\cdot\det\begin{bmatrix}0.30&0.50\\0.10&0.20\end{bmatrix}\end{split}\]
\[= 1(0.20 - 0.04) - 1(0.12 - 0.02) + 1(0.06 - 0.05)\]
\[= 0.16 - 0.10 + 0.01 = \mathbf{0.07}\]
